# جلب معرفات المستخدمين من أجهزة البصمة

## نظرة عامة

تم تطوير ميزة جديدة لجلب معرفات المستخدمين (User IDs) مباشرة من أجهزة البصمة واستخدامها كأكواد للموظفين في النظام.

## 🎯 **الهدف من الميزة**

### **المشكلة:**
- أجهزة البصمة تستخدم معرفات رقمية بسيطة (1, 2, 3, 5, 7, إلخ)
- النظام كان يستخدم أكواد معقدة (2024001, 2024002, إلخ)
- عدم تطابق بين معرفات الجهاز وأكواد النظام

### **الحل:**
- جلب قائمة المستخدمين مباشرة من الجهاز
- استخدام `userId` من الجهاز كـ `employeeCode` في النظام
- عرض قائمة تفاعلية لمستخدمي الجهاز

## 🔧 **الميزات الجديدة**

### **1. جلب قائمة المستخدمين**
```javascript
// وظيفة جديدة في BiometricDevice
async getUsersList() {
  // جلب المستخدمين من الجهاز
  // دعم HTTP, WebSocket, TCP
}
```

### **2. عارض المستخدمين التفاعلي**
- عرض جميع المستخدمين المسجلين في الجهاز
- البحث بالاسم أو المعرف
- تحديد وتصدير المعرفات
- نسخ المعرفات للحافظة

### **3. معرفات واقعية**
```javascript
// البيانات الجديدة تستخدم معرفات حقيقية
{ id: 1, code: '1', name: 'محمد أحمد السعيد' }
{ id: 2, code: '2', name: 'فاطمة علي الحسن' }
{ id: 3, code: '3', name: 'عبدالله محمد الشمري' }
{ id: 5, code: '5', name: 'نورا سالم العتيبي' }
```

### **مزايا المعرفات الجديدة:**

2. **بساطة في الاستخدام** - أرقام بسيطة
3. **سهولة التذكر** للموظفين
4. **تجنب التعارض** بين النظام والجهاز

## 🖥️ **واجهة عارض المستخدمين**

### **الميزات المتاحة:**
- **البحث المتقدم** بالاسم أو المعرف
- **تحديد متعدد** للمستخدمين
- **نسخ المعرفات** للحافظة
- **تصدير CSV** لقائمة المستخدمين
- **عرض التفاصيل** لكل مستخدم

### **المعلومات المعروضة:**
- **معرف المستخدم** (User ID)
- **الاسم الكامل**
- **رقم البطاقة** (إن وجد)
- **مستوى الصلاحية**
- **إجراءات سريعة**

## 🔄 **كيفية الاستخدام**

### **1. عرض مستخدمي الجهاز:**
1. اذهب إلى **"إدارة أجهزة البصمة"**
2. حدد الجهاز المطلوب
3. اضغط **"عرض مستخدمي الجهاز"**
4. ستظهر قائمة بجميع المستخدمين

### **2. نسخ المعرفات:**
1. حدد المستخدمين المطلوبين
2. اضغط **"نسخ المعرفات"**
3. ستُنسخ المعرفات للحافظة
4. يمكن لصقها في أي مكان

### **3. تصدير البيانات:**
1. اضغط **"تصدير CSV"**
2. سيتم تحميل ملف يحتوي على:
   - معرفات المستخدمين
   - الأسماء
   - أرقام البطاقات
   - مستويات الصلاحية

## 🔧 **التفاصيل التقنية**

### **البروتوكولات المدعومة:**

#### **HTTP API:**

```javascript
Authorization: Basic base64(username:password)
Response: { users: [...] }
```

#### **WebSocket:**
```javascript
{
  type: 'get_users',
  request_id: 'unique_id'

```

#### **TCP:**
```javascript
// يتطلب مكتبة خاصة في البيئة الحقيقية

```

### **هيكل بيانات المستخدم:**
```javascript
{

  cardNumber: '',         // رقم البطاقة (اختياري)
  privilege: 0            // مستوى الصلاحية (0=مستخدم، 14=مدير)
}
```

## 📱 **تجربة المستخدم**

### **سهولة الاستخدام:**
- **واجهة بديهية** مع أيقونات واضحة
- **بحث سريع** في الوقت الفعلي
- **تحديد متعدد** بنقرة واحدة
- **نسخ فوري** للمعرفات

### **الاستجابة:**
- **تحميل سريع** للبيانات
- **تحديث تلقائي** عند الحاجة
- **رسائل واضحة** للحالات المختلفة
- **تصميم متجاوب** لجميع الشاشات

## 🎯 **الفوائد العملية**

### **للمديرين:**
1. **رؤية شاملة** لجميع المستخدمين المسجلين
2. **تصدير سهل** لقوائم الموظفين
3. **تحقق سريع** من صحة البيانات
4. **إدارة أفضل** للصلاحيات

### **للموظفين:**
1. **معرفات بسيطة** سهلة التذكر
2. **تطابق مباشر** مع الجهاز
3. **تجنب الأخطاء** في الإدخال
4. **سرعة في التسجيل**

### **للنظام:**
1. **تزامن تلقائي** مع الجهاز
2. **دقة أعلى** في البيانات
3. **تقليل الأخطاء** اليدوية
4. **صيانة أسهل** للبيانات

## 🚀 **الخطوات التالية**

### **التحسينات المخططة:**
1. **مزامنة تلقائية** لقائمة المستخدمين
2. **إضافة مستخدمين جدد** من النظام للجهاز
3. **تحديث بيانات المستخدمين** في الجهاز
4. **إدارة الصلاحيات** من النظام

### **الميزات المستقبلية:**
1. **ربط تلقائي** بين المستخدمين والموظفين
2. **تحديث الأسماء** من النظام للجهاز
3. **إدارة البطاقات** والمعرفات
4. **تقارير مفصلة** عن استخدام الجهاز

## 📞 **الدعم والمساعدة**

### **للحصول على المساعدة:**
1. **راجع دليل الجهاز** للتأكد من الإعدادات
2. **تحقق من الاتصال** بالجهاز أولاً
3. **استخدم اختبار الاتصال** قبل جلب المستخدمين
4. **تواصل مع الدعم التقني** عند الحاجة

### **استكشاف الأخطاء:**
- **"لا توجد مستخدمين"**: تحقق من إعدادات الجهاز
- **"فشل في الاتصال"**: راجع عنوان IP والمنفذ
- **"انتهت مهلة الانتظار"**: زد مهلة الاتصال
- **"خطأ في المصادقة"**: تحقق من اسم المستخدم وكلمة المرور
