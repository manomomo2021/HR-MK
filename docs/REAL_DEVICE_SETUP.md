# دليل إعداد الأجهزة الحقيقية

## نظرة عامة

هذا الدليل يوضح كيفية إعداد النظام للعمل مع أجهزة البصمة الحقيقية بدلاً من البيانات التجريبية.

## متطلبات الشبكة

### 1. إعداد الشبكة
- تأكد من أن جهاز البصمة متصل بنفس الشبكة المحلية
- احصل على عنوان IP الثابت للجهاز
- تأكد من أن المنفذ المطلوب مفتوح (عادة 4370 للـ TCP)

### 2. اختبار الاتصال الأساسي
```bash
# اختبار ping
ping 192.168.0.201

# اختبار المنفذ (Windows)
telnet 192.168.0.201 4370

# اختبار المنفذ (Linux/Mac)
nc -zv 192.168.0.201 4370
```

## البروتوكولات المدعومة

### 1. TCP (الافتراضي)
- الأكثر شيوعاً مع أجهزة ZKTeco
- يتطلب مكتبة خاصة في البيئة الحقيقية
- المنفذ الافتراضي: 4370

### 2. HTTP API
- للأجهزة التي تدعم REST API
- المنفذ الافتراضي: 80 أو 8080
- يتطلب مصادقة (username/password)

### 3. WebSocket
- للاتصال المباشر والتحديثات الفورية
- المنفذ الافتراضي: 8080
- يدعم الاتصال المستمر

## إعداد الجهاز

### 1. إعدادات الشبكة في الجهاز
```
IP Address: 192.168.0.201
Subnet Mask: 255.255.255.0
Gateway: 192.168.0.1
DNS: 8.8.8.8
```

### 2. إعدادات الأمان
```
Username: admin
Password: 123456 (غير الافتراضي!)
Port: 4370
Timeout: 5 seconds
```

### 3. إعدادات الاتصال
```
Protocol: TCP/IP
Communication Password: (إذا كان مطلوباً)
Encryption: (حسب الحاجة)
```

## إعداد النظام

### 1. إضافة جهاز جديد
1. اذهب إلى "إدارة أجهزة البصمة"
2. اضغط "إضافة جهاز"
3. أدخل المعلومات التالية:
   - **الاسم**: اسم وصفي للجهاز
   - **البروتوكول**: اختر TCP للأجهزة التقليدية
   - **عنوان IP**: عنوان الجهاز في الشبكة
   - **المنفذ**: 4370 (أو حسب إعدادات الجهاز)
   - **معرف الجهاز**: معرف فريد
   - **اسم المستخدم**: admin (أو حسب إعدادات الجهاز)
   - **كلمة المرور**: كلمة مرور الجهاز
   - **مهلة الاتصال**: 5-10 ثواني

### 2. اختبار الاتصال
1. حدد الجهاز من القائمة
2. اضغط "اختبار الاتصال"
3. راقب سجلات الاختبار للتأكد من نجاح الاتصال
4. تحقق من معلومات الجهاز المعروضة

### 3. استيراد البيانات
1. بعد نجاح الاختبار، اضغط "استيراد البيانات"
2. اختر "استيراد من تاريخ محدد" لتحديد فترة معينة
3. راقب شريط التقدم
4. تحقق من البيانات في صفحة الحضور والانصراف

## استكشاف الأخطاء

### مشاكل الاتصال الشائعة

#### 1. "انتهت مهلة الاتصال"
**الأسباب المحتملة:**
- عنوان IP خاطئ
- الجهاز غير متصل بالشبكة
- الجدار الناري يحجب الاتصال
- المنفذ خاطئ

**الحلول:**
```bash
# تحقق من الاتصال
ping 192.168.0.201

# تحقق من المنفذ
nmap -p 4370 192.168.0.201

# تعطيل الجدار الناري مؤقتاً للاختبار
```

#### 2. "فشل في المصادقة"
**الأسباب المحتملة:**
- اسم المستخدم أو كلمة المرور خاطئة
- الجهاز لا يتطلب مصادقة
- إعدادات الأمان في الجهاز

**الحلول:**
- تحقق من إعدادات المصادقة في الجهاز
- جرب المعلومات الافتراضية (admin/123456)
- راجع دليل الجهاز للمعلومات الصحيحة

#### 3. "البروتوكول غير مدعوم"
**الأسباب المحتملة:**
- الجهاز لا يدعم البروتوكول المحدد
- إعدادات البروتوكول في الجهاز مختلفة

**الحلول:**
- جرب بروتوكولات مختلفة (TCP → HTTP → WebSocket)
- تحقق من دليل الجهاز للبروتوكولات المدعومة
- تحقق من إعدادات الاتصال في الجهاز

### أخطاء البيانات

#### 1. "لا توجد بيانات"
**الأسباب المحتملة:**
- لا توجد سجلات في الجهاز
- فلترة التاريخ تستبعد جميع السجلات
- مشكلة في قراءة البيانات

**الحلول:**
- تحقق من وجود سجلات في الجهاز مباشرة
- جرب بدون فلترة التاريخ
- تحقق من إعدادات الجهاز

#### 2. "بيانات غير صحيحة"
**الأسباب المحتملة:**
- تنسيق البيانات مختلف عن المتوقع
- إصدار firmware مختلف
- إعدادات الجهاز

**الحلول:**
- تحقق من إصدار firmware الجهاز
- راجع تنسيق البيانات في دليل الجهاز
- تحديث إعدادات معالجة البيانات

## الأمان والحماية

### 1. حماية الشبكة
- استخدم شبكة منفصلة للأجهزة (VLAN)
- قم بتشفير الاتصال إذا كان مدعوماً
- استخدم VPN للوصول عن بُعد

### 2. حماية البيانات
- غير كلمات المرور الافتراضية
- فعل التشفير في الجهاز
- قم بنسخ احتياطية منتظمة

### 3. مراقبة الوصول
- سجل جميع عمليات الاتصال
- راقب المحاولات الفاشلة
- استخدم صلاحيات محدودة

## الصيانة والمراقبة

### 1. المراقبة المستمرة
- تحقق من حالة الاتصال دورياً
- راقب مساحة التخزين في الجهاز
- تحقق من دقة الوقت والتاريخ

### 2. الصيانة الدورية
- نظف ذاكرة الجهاز من السجلات القديمة
- حدث firmware الجهاز
- تحقق من إعدادات الشبكة

### 3. النسخ الاحتياطية
- اعمل نسخة احتياطية من إعدادات الجهاز
- احفظ قاعدة بيانات المستخدمين
- اعمل نسخة احتياطية من السجلات

## التطوير المتقدم

### 1. للمطورين
إذا كنت تريد تطوير دعم أفضل للأجهزة:

```javascript
// مثال لاستخدام مكتبة node-zk (Node.js)
const ZK = require('node-zk')

const device = new ZK('192.168.0.201', 4370, 5000)
await device.connect()
const logs = await device.getAttendances()
await device.disconnect()
```

### 2. API مخصص
يمكنك إنشاء API وسيط للتعامل مع الأجهزة:

```javascript
// server.js
app.post('/api/biometric/connect', async (req, res) => {
  // منطق الاتصال بالجهاز
})

app.get('/api/biometric/logs', async (req, res) => {
  // منطق جلب السجلات
})
```
