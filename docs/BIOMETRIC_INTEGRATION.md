# دليل تكامل أجهزة البصمة - الإصدار المحدث

## نظرة عامة

تم تطوير نظام متكامل ومحدث للاتصال بأجهزة البصمة الحقيقية وجلب بيانات الحضور والانصراف تلقائياً. النظام يدعم بروتوكولات متعددة (TCP, HTTP, WebSocket) ويوفر واجهة متقدمة لإدارة الأجهزة واستيراد البيانات مع اختبار الاتصال المتقدم.

## الميزات الرئيسية

### 1. إدارة أجهزة البصمة المتقدمة
- إضافة وتحرير وحذف أجهزة البصمة
- دعم بروتوكولات متعددة (TCP, HTTP, WebSocket)
- اختبار الاتصال المتقدم مع سجلات مفصلة
- مراقبة حالة الأجهزة في الوقت الفعلي
- حفظ إعدادات الأجهزة محلياً
- دعم المصادقة (username/password)

### 2. استيراد البيانات الذكي
- جلب سجلات الحضور والانصراف من الأجهزة الحقيقية
- تصفية البيانات حسب التاريخ
- إزالة السجلات المكررة تلقائياً
- معالجة أنواع السجلات المختلفة (دخول/خروج)
- شريط تقدم مع تفاصيل العملية
- إعادة المحاولة التلقائية عند فشل الاتصال

### 3. معالجة البيانات المتطورة
- تحويل أكواد الحالة إلى أنواع مفهومة
- تنظيف البيانات وإزالة التكرارات
- ربط السجلات بالموظفين تلقائياً
- حفظ البيانات في النظام مع الطوابع الزمنية
- دعم العملات المتعددة في الرواتب

### 4. واجهة المستخدم المحسنة
- اختبار الاتصال التفاعلي مع سجلات مفصلة
- مؤشرات حالة الأجهزة في الوقت الفعلي
- إعدادات متقدمة لكل جهاز
- رسائل خطأ واضحة ونصائح الإصلاح

## التكوين

### إعدادات الجهاز الافتراضية

```javascript
const DEFAULT_DEVICE_CONFIG = {
  ip: '192.168.0.201',        // عنوان IP للجهاز
  port: 4370,                 // منفذ الاتصال
  deviceId: 'iface7',         // معرف الجهاز
  timeout: 5000,              // مهلة الاتصال (بالميلي ثانية)
  retryAttempts: 3,           // عدد محاولات إعادة الاتصال
  retryDelay: 2000            // تأخير بين المحاولات
}
```

### أكواد الحالة

| كود الحالة | النوع | الوصف |
|------------|-------|--------|
| 0, 1 | IN | حضور/دخول |
| 4, 5 | OUT | انصراف/خروج |
| أخرى | IN | افتراضي (حضور) |

## الاستخدام

### 1. إضافة جهاز جديد

```javascript
// إنشاء مثيل جديد من جهاز البصمة
const device = new BiometricDevice({
  ip: '192.168.1.100',
  port: 4370,
  deviceId: 'device_1',
  name: 'جهاز البصمة الرئيسي'
})
```

### 2. اختبار الاتصال

```javascript
// اختبار الاتصال بالجهاز
const result = await device.testConnection()
if (result.success) {
  console.log('تم الاتصال بنجاح')
  console.log('معلومات الجهاز:', result.info)
} else {
  console.error('فشل الاتصال:', result.error)
}
```

### 3. جلب سجلات الحضور

```javascript
// الاتصال بالجهاز
await device.connect()

// جلب جميع السجلات
const allLogs = await device.getAttendanceLogs()

// جلب السجلات من تاريخ محدد
const recentLogs = await device.getAttendanceLogs('2024-01-01')

// قطع الاتصال
await device.disconnect()
```

### 4. معالجة السجلات

```javascript
// تحويل كود الحالة إلى نوع السجل
const logType = mapStatusToLogType(1) // 'IN'

// الحصول على النص العربي
const arabicText = getLogTypeText('IN') // 'حضور'
```

## استكشاف الأخطاء

2. تأكد من وجود الموظفين في النظام
3. راجع سجلات الأخطاء
4. تحقق من صحة أكواد الحالة
